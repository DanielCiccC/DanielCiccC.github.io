12.2 Merge-Sort

Divide and Conquer

The first two algorithms in this chapter use recursion in an algorithmic design patterns called divide and conquer. This pattern consists of the following three steps:

  1. Divide: If the input size is smaller than a certain threshold, solve the problem directly using a straightforward method and return the solution so obtained. Otherwise, divide the input data into two or more disjoint subsets
  2. Conquer: Recursively solve the subproblems associated with the subsets
  3. Combine: Take the solutions to the subproblems and merge them into a solution to the original problem.

Implementing into an algorithm:

  1. Divide: If SSS has zero or one elements, return SSS (as it is already sorted). Otherwise, remove all the elements from SSS and put them into two sequences, S1S_{1}S1​ and S2S_{2}S2​, each containing about half of the elements of S.
  2. Conquer: Recursively sort sequences of S1S_{1}S1​ and S2S_{2}S2​.
  3. Combine: Put back the elements into SSS by merging the sorted sequences S1S_{1}S1​ and S2S_{2}S2​ into a sorted sequence, i.e. ⌊n/2⌋\lfloor n/2 \rfloor⌊n/2⌋ elements of S1S_{1}S1​ and ⌈n/2⌉\lceil n/2 \rceil⌈n/2⌉ elements of S2S_{2}S2​.

Implementation of Merge-Sort

  • The merge function is responsible for the subtask of merging two previously sorted sequences, S1S_{1}S1​ and S2S_{2}S2​, with the output copied into SSS.
    • We copy one element during each pass of the while loop, conditionally determining whether the next elements should be taken from S1S_{1}S1​ or S2S_{2}S2​.
def merge(S1, S2, S):
    """Merge two sorted Python lists S1 and S2 into properly sized list S"""
    i = j = 0
    while i + j < len(S):
        if j == len(S2) or (i < len(S1) and S1[i] < S2[j]):
            S[i + j] = S1[i]
            i += 1
        else:
            S[i + j] = S2[j]
            j += 1

def merge_sort(S):
    """Sort the elements of Python list S using the merge-sort algorithm."""
    n = len(S)
    if n < 2:
        return # list is already sorted
    # divide
    mid = n // 2
    S1 = S[0:mid] # copy of ﬁrst half
    S2 = S[mid:n] # copy of second half
    # conquer (with recursion)
    merge_sort(S1) # sort copy of ﬁrst half
    merge_sort(S2) # sort copy of second half
    # merge results
    merge(S1, S2, S) # merge sorted halves back into S

Running time of Merge-Sort

Let n1n_{1}n1​ and n2n_{2}n2​ be the number of elements of S1S_{1}S1​ and S2S_{2}S2​, respectively. It is clear that the operations performed inside each pass of the while loop (of function merge) take O(1)O(1)O(1) time.

  • During each iteration of the loop, one element is copied from either s1s_{1}s1​ or S2S_{2}S2​ into SSS (and the element is considered no further). Therefore, the number of iterations of the loop is n1+n2n_{1} + n_{2}n1​+n2​ (and the time complexity would be O(n1+n2)O(n_{1} + n_{2})O(n1​+n2​)).

To determine the the complexity of the merge-sort algorithm, we account for the time to divide the sequence into two subsequences, and the call to merge to combine the two sorted sequences, but we exclude the two recursive calls to merge-sort.

Observations:

  • The divide step at node vvv runs in time proportional to the size of the sequence for vvv
  • The merging step also takes time that is linear in the size of the merged sequence.
    • If we let iii denote the depth of node vvv, the time spent at node vvv is O(n/2i)O(n/2^{i})O(n/2i) since the size of the sequence handled by the recursive called associated with vvv is equal to n/2in/2^{i}n/2i

Given our definition of 'time spent at a node', the running time of merge-sort is equal to the sum of the times spent at the nodes of TTT. Observe TTT has exactly 2i2^{i}2i nodes at depth iii.

  • The overall time spent at all the nodes of TTT is O(2i⋅n/2i)O(2^{i} \cdot n/2^{i})O(2i⋅n/2i), which is O(n)O(n)O(n).
  • The height of TTT is ⌈log⁡n⌉\lceil \log n \rceil⌈logn⌉ Thus, the algorithm merge-sort sorts a sequence SSS of size nnn in O(nlog⁡n)O(n \log n)O(nlogn) times, assuming two elements can be compared in O(1)O(1)O(1) time.

Quick Sort

High level description of Quick-Sort:

The main idea is to apply a divide-and-conquer technique, and then combine the sorted subsequences by a simple concatenation.
